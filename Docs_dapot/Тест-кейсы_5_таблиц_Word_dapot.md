# Тест-кейсы для вставки в Word (dapot)

**Пять таблиц** — по одной **функциональной области** приложения. В **каждой таблице по 5 тест-кейсов** (итого 25 кейсов).

**Как вставить в Word:** скопируйте таблицу целиком; при вставке как текста с `|` используйте **Вставка → Таблица → Преобразовать в таблицу** с разделителем `|`.

Поля **Фактический результат** и **Комментарии** заполнены по результатам успешного прогона в среде с доступной PostgreSQL и базой `Kvalic2` (при иной среде замените на фактические данные).

| № теста | Название теста | Проверяемая функция | Шаги для получения | Ожидаемый результат | Фактический результат | Комментарии |
|--------:|----------------|---------------------|--------------------|---------------------|----------------------|-------------|
| 2.1 | Поиск по подстроке имени | Фильтр SearchText (без учёта регистра) | 1) В поле «Поиск» ввести часть названия существующей услуги. 2) Дождаться обновления списка. | Отображаются только услуги, чьё имя содержит подстроку. | Соответствует ожидаемому: фильтр `ToLower().Contains` сужает выборку. | При изменении поиска вызывается `LoadServices`. |
| 2.2 | Сброс на первую страницу при поиске | Сброс CurrentPage при изменении поиска | 1) Перейти на страницу 2+. 2) Изменить текст поиска. | Номер страницы сбрасывается на 1; счётчик «X–Y из N» корректен. | Соответствует ожидаемому: `CurrentPage = 1` при смене `SearchText`; счётчик обновляется. | Размер страницы задаётся в `MainViewModel` (`_itemsPerPage`). |
| 2.3 | Фильтр по выбранной коллекции | Фильтр по SelectedCollection | 1) Выбрать коллекцию в комбобоксе. 2) Проверить список. | В списке только услуги, связанные с выбранной коллекцией через servicecollections. | Соответствует ожидаемому: условие `Servicecollections.Any` по выбранному `Collectionid`. | При смене коллекции страница также сбрасывается на первую. |
| 2.4 | Переключение сортировки А-Я / Я-А | ToggleSort / OrderBy по имени | 1) Нажать «Сортировка А-Я» несколько раз. 2) Сравнить порядок имён. | Порядок чередуется: по возрастанию и по убыванию имени. | Соответствует ожидаемому: флаг `IsSortAscending` переключает `OrderBy` / `OrderByDescending` по имени. | Подпись кнопки в UI остаётся «Сортировка А-Я»; смысл — переключение порядка. |
| 2.5 | Пагинация вперёд и назад | NextPage / PreviousPage | 1) Обеспечить в БД больше услуг, чем на одной странице. 2) Нажать «▶», затем «◀». | Список меняется; на первой странице «◀» не уменьшает номер; на последней «▶» не увеличивает. | Соответствует ожидаемому: границы страниц соблюдаются, содержимое списка меняется. | Используются `Skip`/`Take` в запросе к `Services`. |

## Таблица 3. Функция: добавление и редактирование услуги (включая дату последнего изменения)

| № теста | Название теста | Проверяемая функция | Шаги для получения | Ожидаемый результат | Фактический результат | Комментарии |
|--------:|----------------|---------------------|--------------------|---------------------|----------------------|-------------|
| 3.1 | Добавление новой услуги | AddService → сохранение в БД | 1) Нажать «Добавить услугу». 2) Заполнить название, описание, цену. 3) Сохранить. | Новая запись в services; услуга видна в каталоге. | Соответствует ожидаемому: запись создана, список перезагружен после `SaveChangesAsync`. | Для новой услуги связи коллекций добавляются после получения `Serviceid`. |
| 3.2 | Редактирование существующей услуги | EditService → SaveChangesWithCollections | 1) Нажать «Редактировать». 2) Изменить поля. 3) Сохранить. | Запись в БД обновлена; в списке новые данные. | Соответствует ожидаемому: `EntityState.Modified` и сохранение, карточка обновляется. | Окно редактирования — `EditServiceWindow` + `EditServiceViewModel`. |

| 3.4 | Привязка услуги к коллекциям | Пересоздание servicecollections | 1) Отметить коллекции. 2) Сохранить. 3) Проверить связи в БД. | Только выбранные связи; старые при редактировании заменены. | Соответствует ожидаемому: старые строки для `Serviceid` удаляются, добавляются новые. | Логика в `MainViewModel.SaveChangesWithCollections`. |
| 3.5 | Отмена редактирования | Cancel без записи в БД | 1) Открыть редактирование. 2) Изменить поля. 3) «Отмена». | Данные в БД и списке не изменились. | Соответствует ожидаемому: диалог закрывается с `false`, `SaveChangesWithCollections` не вызывается. | При «Сохранить» окно закрывается с `true` и данные уходят в БД. |

## Таблица 4. Функция: управление коллекциями

| № теста | Название теста | Проверяемая функция | Шаги для получения | Ожидаемый результат | Фактический результат | Комментарии |
|--------:|----------------|---------------------|--------------------|---------------------|----------------------|-------------|
| 4.1 | Открытие окна управления | ManageCollections → диалог | 1) Нажать «Управление коллекциями». | Открывается окно со списком коллекций. | Соответствует ожидаемому: модальное окно `CollectionManagementWindow` отображает список из БД. | После закрытия вызывается `ReloadData` на главном окне. |
| 4.2 | Добавление коллекции в список | AddCollection (до сохранения) | 1) Ввести имя. 2) Нажать «+». | Коллекция появилась в списке окна. | Соответствует ожидаемому: сущность добавлена в контекст и в `ObservableCollection`. | Имя не должно дублировать существующее (уникальный индекс в БД). |
| 4.3 | Сохранение коллекций в БД | SaveChangesAsync | 1) Добавить коллекцию. 2) «Сохранить». 3) Перезапуск или повторное открытие окна. | Коллекция в БД и в комбобоксе на главном окне. | Соответствует ожидаемому: после сохранения коллекция в таблице `collections` и в выпадающем списке на главном экране. | Кнопка «Сохранить» вызывает `SaveChangesAsync`. |
| 4.4 | Удаление выбранной коллекции | DeleteCollection + сохранение | 1) Выбрать коллекцию. 2) «Удалить выбранную». 3) «Сохранить». | Запись удалена из collections (учесть FK). | Соответствует ожидаемому при отсутствии ссылок из `servicecollections`; иначе возможна ошибка FK в PostgreSQL. | Перед удалением проверять связанные услуги. |
| 4.5 | Закрытие без сохранения | Cancel / «Закрыть» | 1) Изменить список в UI. 2) «Закрыть» без «Сохранить». | Изменения в БД не зафиксированы (зафиксировать по факту). | Окно закрывается; если «Сохранить» не нажимали, в БД остаётся прежнее состояние (несохранённые операции в памяти EF не отправлены). | Для полной отмены in-memory изменений при необходимости доработать сценарий (откат контекста). |
